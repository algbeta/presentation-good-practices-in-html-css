Наталья Горбунова | n.gorbunova@softteco.com | skype: algbeta

Почему верстать хорошо - сложно?

  1. Большое кол-во браузеров(даже если выкидывать ненавистные IE)
  2. Тяжелая расширяемость проекта
  3. Количество CSS ВСЕГДА увеличивается и его сложно поддерживать
  4. Обычный CSS тяжело понять без контекста
  5. CSS поощряет переопределение свойств и каскад
  6. Свойства одного элемента могут быть описаны в удаленных друг от друга местах

Как должен выглядеть идеальный код?

  1. Должен работать
  2. Должен быть лаконичным => в нем легко разобраться
  3. Быть реиспользуемым
  4. Легко расширяться, сохраняя предыдущие свойства
  5. Автоматизированным, насколько это возможно
  6. Содержимое, стилизация, функционал разделены, насколько это возможно

Методологии верстки

SMACSS

Scalable and Modular Architecture for CSS. Джонатан Снук. 2011. https://smacss.com/

Предлагает разделять все CSS-классы на несколько типов: основа (базис), структура (каркас), модуль (группа элементов), состояние,тема.

Основа(Base)

Это reset стилей и стилизация тэгов

				html, body, form {
				  padding: 0;
				  margin: 0;
			    }
			    a {
			       color: #bbb;
		        }

Структура(Layout)

Классы для указания размеров, обтекания, позиционирования и т.д. Имеют приставку l-* 

				.page-header, .page-footer {
				  width: 940px;
				  margin: 0 auto;
			    }
			    .l-left-column {
			       float: right;
		        }

Модуль(Module)

Более дискретная, независимая от контента часть страницы(виджет, диалог) 

				.module h2{
				  padding: 40px;
				}
https://smacss.com/book/type-module

Состояние(State)

Классы для описания любого состояния, отличного от состояния по умолчанию. Приставка is-* 

				.accodion-header{
				  background-color: #ddd;
				}
				.is-accodion-header-active{
				  background-color: #fff;
				}

Тема(Theme)

Тема — набор правил, определяющих внешнее оформление любых элементов (цвета текста и фона, оформительские изображения).

Методологии верстки

BEM

Block. Element. Modifier.
Оригинал и модификация Галлахера.

Достоинства BEM

  1. Простота в освоении
    1. Нет селектора вне класса
    2. Избегаем каскада
  2. Самодокументироваемость кода
  3. Независимость блоков
    1. нет пересечения имен => однозначная работа кода
    2. легкая поддержка и расширение, использование в разных контекстах
  4. Отпадает проблема с наследованием и сложными селекторами, css менее привязан к html

Shower Key Features

  4. Fully keyboard accessible
